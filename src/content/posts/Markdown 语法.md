## 概述

Markdown 是一种轻量级标记语言，由 John Gruber 在 2004 年创建。它使用易读易写的纯文本格式编写文档，然后转换成有效的 HTML。Markdown 被广泛用于 README 文件、论坛、博客等场景。

## 核心概念

理解 Markdown 的设计哲学：可读性优先，让普通文本和格式文本的区分尽可能自然。Markdown 文件应该以纯文本形式发布，而不是二进制格式。

## 标题

Markdown 支持两种标题语法：

### Setext 风格（下划线）

```markdown
一级标题
==========

二级标题
----------
```

### ATX 风格（# 号）

```markdown
# 一级标题

## 二级标题

### 三级标题

#### 四级标题

##### 五级标题

###### 六级标题
```

**注意：** 一级标题通常用于文档标题，建议每个文档只有一个一级标题。

## 文本格式

### 粗体和斜体

```markdown
*斜体* 或 _斜体_

**粗体** 或 __粗体_

***粗斜体*** 或 ___粗斜体___
```

**效果：**
- *斜体*
- **粗体**
- ***粗斜体***

### 删除线

```markdown
~~删除线文本~~
```

**效果：** ~~删除线文本~~

### 下划线和高亮

```markdown
<u>下划线</u>

==高亮== (某些解析器支持)
```
<u>下划线</u>

## 列表

### 无序列表

使用 `*`、`+` 或 `-` 开头：

```markdown
* 苹果
* 香蕉
* 橙子

+ 苹果
+ 香蕉
+ 橙子

- 苹果
- 香蕉
- 橙子
```

### 有序列表

使用数字加点号：

```markdown
1. 第一步
2. 第二步
3. 第三步
```

**注意：** Markdown 会自动处理编号，你可以统一使用 `1.`：

```markdown
1. 第一步
1. 第二步
1. 第三步
```

### 嵌套列表

```markdown
1. 水果
   - 苹果
   - 香蕉
2. 蔬菜
   - 西红柿
   - 黄瓜
```

**效果：**
1. 水果
   - 苹果
   - 香蕉
2. 蔬菜
   - 西红柿
   - 黄瓜

### 任务列表

GitHub Flavored Markdown (GFM) 扩展语法：

```markdown
- [x] 已完成的任务
- [ ] 未完成的任务
- [ ] 另一个待办事项
```

**效果：**
- [x] 已完成的任务
- [ ] 未完成的任务
- [ ] 另一个待办事项

## 链接

### 行内链接

```markdown
[链接文本](https://example.com)

[带标题的链接](https://example.com "鼠标悬停提示")
```

### 参考式链接

```markdown
[链接文本][id]

[id]: https://example.com "可选标题"
```

### 自动链接

```markdown
<https://example.com>
<email@example.com>
```

## 图片

图片语法与链接类似，前面加 `!`：

```markdown
![替代文本](https://example.com/image.jpg)

![带标题的图片](https://example.com/image.jpg "图片标题")
```

### 参考式图片

```markdown
![替代文本][img-id]

[img-id]: https://example.com/image.jpg "图片标题"
```

## 代码

### 行内代码

使用反引号 \` 包裹：

```markdown
使用 `console.log()` 输出信息
```

### 代码块

使用三个反引号包裹，可指定语言：

````
```javascript
function hello() {
  console.log('Hello, World!');
}
```
````

**效果：**
```javascript
function hello() {
  console.log('Hello, World!');
}
```

### 缩进代码块（4个空格或1个Tab）

```markdown
    function hello() {
      console.log('Hello, World!');
    }
```

## 引用

使用 `>` 符号：

```markdown
> 这是一段引用
>
> 可以包含多行
>
>> 嵌套引用
```

**效果：**
> 这是一段引用
>
> 可以包含多行
>
>> 嵌套引用

## 表格

### 基本表格

```markdown
| 表头1 | 表头2 | 表头3 |
|-------|-------|-------|
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |
```

### 对齐方式

```markdown
| 左对齐 | 居中 | 右对齐 |
|:-------|:----:|-------:|
| 文本   | 文本 | 文本   |
| 文本   | 文本 | 文本   |
```

**效果：**

| 左对齐 | 居中 | 右对齐 |
|:-------|:----:|-------:|
| 文本   | 文本 | 文本   |
| 文本   | 文本 | 文本   |

## 分隔线

使用三个或更多的 `*`、`-` 或 `_`：

```markdown
***

---

___
```

**效果：**

***

---

___

## 转义字符

使用反斜杠 `\` 转义特殊字符：

```markdown
\*不是斜体\*

\[不是链接\]

\`不是代码\`
```

## HTML 支持

Markdown 支持内联 HTML：

```markdown
<div style="color: red;">红色文本</div>

<details>
  <summary>点击展开</summary>
  隐藏内容
</details>
```

## 高级语法


### 定义列表


## 注意事项

1. **空格很重要**：Markdown 的许多语法都需要在符号后加空格
2. **标题层级**：不要跳级使用标题（如从一级标题直接跳到三级）
3. **列表缩进**：嵌套列表需要正确的缩进
4. **链接文本**：链接文本应该简短而有意义
5. **代码语言**：指定代码块的语言以获得更好的高亮
6. **图片替代文本**：为图片提供有意义的替代文本
7. **表格对齐**：注意表格的对齐符号 `:` 的位置
8. **转义字符**：需要显示特殊符号时使用转义

## 最佳实践

1. **保持简洁**：优先使用简单语法，避免过度复杂化
2. **可读性**：源代码本身应该易于阅读和理解
3. **一致性**：在整个项目中保持一致的 Markdown 风格
4. **结构化**：合理使用标题层级，建立清晰的文档结构
5. **示例代码**：提供可运行的代码示例
6. **链接检查**：确保所有链接都是有效的
7. **图片优化**：使用适当大小和格式的图片
8. **版本控制**：将 Markdown 文件纳入版本控制

通过本文的学习，相信你已经对 Markdown 语法有了更深入的理解。Markdown 的简洁性和可读性使其成为技术写作和文档编写的理想选择。掌握 Markdown 的基本语法和最佳实践，可以帮助你高效地编写出专业、易读的文档。
