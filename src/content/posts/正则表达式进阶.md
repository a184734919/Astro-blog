## 概述

正则表达式是处理字符串的强大工具，它使用特定的模式来匹配、查找、替换文本。进阶的正则表达式技巧能够帮助我们更高效、更准确地处理复杂的字符串操作。本文将深入探讨正则表达式的高级特性、性能优化和实际应用。

## 基础回顾

### 基本语法

```javascript
// 字符匹配
const pattern1 = /abc/; // 匹配 "abc"
const pattern2 = /[abc]/; // 匹配 a、b 或 c
const pattern3 = /[^abc]/; // 匹配除 a、b、c 外的任意字符
const pattern4 = /[a-z]/; // 匹配任意小写字母
const pattern5 = /[A-Z]/; // 匹配任意大写字母
const pattern6 = /[0-9]/; // 匹配任意数字
const pattern7 = /\w/; // 匹配单词字符（字母、数字、下划线）
const pattern8 = /\d/; // 匹配数字
const pattern9 = /\s/; // 匹配空白字符
const pattern10 = ./; // 匹配除换行外的任意字符

// 量词
const pattern11 = /a+/; // 匹配 1 个或多个 a
const pattern12 = /a*/; // 匹配 0 个或多个 a
const pattern13 = /a?/; // 匹配 0 个或 1 个 a
const pattern14 = /a{3}/; // 匹配恰好 3 个 a
const pattern15 = /a{3,5}/; // 匹配 3 到 5 个 a
const pattern16 = /a{3,}/; // 匹配 3 个或更多 a

// 边界匹配
const pattern17 = /^abc/; // 匹配以 abc 开头
const pattern18 = /abc$/; // 匹配以 abc 结尾
const pattern19 = /\bword\b/; // 匹配单词边界
const pattern20 = /\B/; // 匹配非单词边界
```

### 基本方法

```javascript
// test() - 测试是否匹配
const regex1 = /hello/;
console.log(regex1.test('hello world')); // true

// exec() - 执行搜索
const regex2 = /hello/;
const result = regex2.exec('hello world');
console.log(result); // ['hello', index: 0, input: 'hello world', groups: undefined]

// match() - 字符串匹配
const text = 'hello world hello';
const matches = text.match(/hello/g);
console.log(matches); // ['hello', 'hello']

// matchAll() - 匹配所有（ES2020）
const allMatches = text.matchAll(/hello/g);
console.log([...allMatches]); // 匹配结果的数组

// replace() - 替换
const replaced = 'hello world'.replace(/hello/, 'hi');
console.log(replaced); // 'hi world'

// replace() 使用回调函数
const replaced2 = 'hello world hello'.replace(/hello/g, (match, offset, string) => {
  return match.toUpperCase();
});
console.log(replaced2); // 'HELLO world HELLO'

// split() - 分割
const parts = 'a,b,c,d'.split(/,/);
console.log(parts); // ['a', 'b', 'c', 'd']

// search() - 搜索位置
const index = 'hello world'.search(/world/);
console.log(index); // 6
```

## 捕获组

### 基本捕获组

```javascript
// 捕获组使用括号 ()
const regex1 = /(\w+)\s+(\w+)/;
const text = 'John Doe';
const result1 = regex1.exec(text);

console.log(result1);
// ['John Doe', 'John', 'Doe', index: 0, input: 'John Doe', groups: undefined]

console.log(result1[1]); // 'John' - 第一个捕获组
console.log(result1[2]); // 'Doe' - 第二个捕获组
console.log(result1[0]); // 'John Doe' - 完整匹配

// 使用 $1, $2 等引用捕获组
const replaced = 'John Doe'.replace(/(\w+)\s+(\w+)/, '$2, $1');
console.log(replaced); // 'Doe, John'

// 在回调函数中使用捕获组
const replaced2 = 'John Doe'.replace(/(\w+)\s+(\w+)/, (match, first, last) => {
  return `${last}, ${first}`;
});
console.log(replaced2); // 'Doe, John'

// 捕获组编号
const regex2 = /(\w+)-(\w+)-(\w+)/;
const result2 = regex2.exec('one-two-three');

console.log(result2[1]); // 'one'
console.log(result2[2]); // 'two'
console.log(result2[3]); // 'three'
```

### 命名捕获组

```javascript
// 使用 (?<name>) 创建命名捕获组
const regex = /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/;
const text = '2024-01-15';
const result = regex.exec(text);

console.log(result);
// ['2024-01-15', '2024', '01', '15', index: 0, input: '2024-01-15', groups: { year: '2024', month: '01', day: '15' }]

// 访问命名捕获组
console.log(result.groups.year); // '2024'
console.log(result.groups.month); // '01'
console.log(result.groups.day); // '15'

// 在 replace 中使用命名捕获组
const replaced = '2024-01-15'.replace(
  /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/,
  '$<day>/$<month>/$<year>'
);
console.log(replaced); // '15/01/2024'

// 在回调函数中使用命名捕获组
const replaced2 = '2024-01-15'.replace(
  /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/,
  (match, year, month, day, offset, string, groups) => {
    return `${groups.day}/${groups.month}/${groups.year}`;
  }
);
console.log(replaced2); // '15/01/2024'

// 实际应用：解析 URL
const urlRegex = /^https?:\/\/(?<domain>[^/]+)(?<path>\/.*)?$/;
const url = 'https://example.com/path/to/page';
const urlResult = urlRegex.exec(url);

console.log(urlResult.groups.domain); // 'example.com'
console.log(urlResult.groups.path); // '/path/to/page'
```

### 非捕获组

```javascript
// 使用 (?:) 创建非捕获组，只用于分组但不捕获
const regex1 = /(?:http|https):\/\/(.+)/;
const text = 'https://example.com';
const result1 = regex1.exec(text);

console.log(result1);
// ['https://example.com', 'example.com', index: 0, input: 'https://example.com', groups: undefined]

// 对比捕获组
const regex2 = /(http|https):\/\/(.+)/;
const result2 = regex2.exec(text);

console.log(result2);
// ['https://example.com', 'https', 'example.com', index: 0, input: 'https://example.com', groups: undefined]

console.log(result1.length); // 2 (完整匹配 + 1个捕获组)
console.log(result2.length); // 3 (完整匹配 + 2个捕获组)

// 实际应用：匹配多种格式但只捕获部分
const dateRegex = /(?:\d{4}-\d{2}-\d{2})|(?:\d{2}\/\d{2}\/\d{4})/;
const date1 = '2024-01-15';
const date2 = '01/15/2024';

console.log(dateRegex.test(date1)); // true
console.log(dateRegex.test(date2)); // true
```

## 零宽断言

### 正向先行断言

```javascript
// (?=pattern) - 匹配后面是 pattern 的位置
const regex1 = /\d+(?=%)/;
const text1 = '50% off, 100%';

const matches1 = text1.match(regex1);
console.log(matches1); // ['50', '100']

// 实际应用：匹配后面跟着特定单位的价格
const priceRegex = /\$\d+(?=\s*(USD|EUR))/;
const priceText = 'Price: $100 USD, $200 EUR';

const prices = priceText.matchAll(priceRegex);
console.log([...prices].map(m => m[0])); // ['$100', '$200']

// 使用命名捕获组
const pattern = /\$(?<amount>\d+)(?=\s*(?<currency>USD|EUR))/;
const result = pattern.exec('$100 USD');

console.log(result.groups.amount); // '100'
console.log(result.groups.currency); // 'USD'
```

### 正向后行断言

```javascript
// (?<=pattern) - 匹配前面是 pattern 的位置
const regex2 = /(?<=\$)\d+/;
const text2 = '$100, $200, $300';

const matches2 = text2.match(regex2);
console.log(matches2); // ['100', '200', '300']

// 实际应用：匹配美元符号后的数字
const moneyRegex = /(?<=\$)\d+(?=\s*(USD|EUR))/;
const moneyText = 'I have $100 USD and $200 EUR';

const amounts = moneyText.matchAll(moneyRegex);
console.log([...amounts].map(m => m[0])); // ['100', '200']

// 匹配特定属性值
const attrRegex = /(?<=value=")[^"]+(?=")/;
const html = '<input value="test1"><input value="test2">';

const values = html.matchAll(attrRegex);
console.log([...values].map(m => m[0])); // ['test1', 'test2']
```

### 负向先行断言

```javascript
// (?!pattern) - 匹配后面不是 pattern 的位置
const regex3 = /\b\w+(?!\w)/; // 匹配单词的最后一个字符
const text3 = 'hello world';

const matches3 = text3.match(regex3);
console.log(matches3); // ['o', 'd']

// 实际应用：不匹配特定字符后的数字
const regex4 = /\d+(?!\d)/; // 匹配不后面跟着数字的数字
const text4 = '123 456 789';

const matches4 = text4.match(regex4);
console.log(matches4); // ['3', '6', '9']

// 排除特定单词
const regex5 = /\b\w+\b(?!\s+test)/;
const text5 = 'hello test world sample';

const matches5 = text5.match(regex5);
console.log(matches5); // ['hello', 'sample']

// 实际应用：验证密码不包含特定字符
const passwordRegex = /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?!.*[!@#$%^&*]).{8,}$/;
console.log(passwordRegex.test('Password123')); // true
console.log(passwordRegex.test('Password123!')); // false
```

### 负向后行断言

```javascript
// (?<!pattern) - 匹配前面不是 pattern 的位置
const regex6 = /\d+(?<!\$)/; // 匹配不在 $ 符号后的数字
const text6 = '$100, 200, $300';

const matches6 = text6.match(regex6);
console.log(matches6); // ['200']

// 实际应用：匹配不在标签内的文本
const tagRegex = /(?<!<[^>]*)>/;
const htmlText = '<p>test</p><div>content</div>';

// 注意：这在某些正则引擎中可能不支持，JavaScript 的支持有限

// 验证手机号不以特定前缀开头
const phoneRegex = /^(?!0|100)\d{11}$/;
console.log(phoneRegex.test('13800138000')); // true
console.log(phoneRegex.test('010012345678')); // false
```

## 反向引用

### 基本反向引用

```javascript
// \1, \2 等引用前面的捕获组
const regex1 = /(\w+)\s+\1/;
const text1 = 'hello hello world world test test';

const matches1 = text1.match(regex1);
console.log(matches1); // ['hello hello', 'hello']

// 实际应用：匹配重复单词
const repeatRegex = /\b(\w+)\b\s+\1\b/g;
const repeatText = 'this is is a test test case';

const repeats = repeatText.match(repeatRegex);
console.log(repeats); // ['is is', 'test test']

// 使用反向引用替换
const replaced = 'is is a test test'.replace(/\b(\w+)\b\s+\1\b/g, '$1');
console.log(replaced); // 'is a test'

// 匹配 HTML 标签对
const tagRegex = /<(\w+)[^>]*>.*?<\/\1>/;
const html = '<div>content</div><span>text</span>';

const tags = html.match(tagRegex);
console.log(tags); // ['<div>content</div>']
```

### 命名反向引用

```javascript
// \k<name> 引用命名捕获组
const regex2 = /(?<word>\w+)\s+\k<word>/;
const text2 = 'hello hello world';

const result2 = regex2.exec(text2);
console.log(result2[0]); // 'hello hello'
console.log(result2.groups.word); // 'hello'

// 实际应用：验证标签闭合
const htmlRegex = /<(?<tag>\w+)[^>]*>.*?<\/\k<tag>>/;
const html1 = '<div>content</div>';
const html2 = '<div>content</span>';

console.log(htmlRegex.test(html1)); // true
console.log(htmlRegex.test(html2)); // false

// 复杂示例：匹配引号内的内容
const quoteRegex = /(?<quote>["'])(?<content>.*?)\k<quote>/;
const quoteText = 'He said "hello" and she said \'hi\'';

const quotes = quoteText.matchAll(quoteRegex);
console.log([...quotes].map(m => m.groups.content)); // ['hello', 'hi']
```

### 实际应用示例

```javascript
// 1. 验证日期格式：YYYY-MM-DD（简化版）
const dateRegex = /^(?<year>\d{4})-(?<month>\d{2})-\k<year>/;
console.log(dateRegex.test('2024-01-2024')); // true
console.log(dateRegex.test('2024-01-2025')); // false

// 2. 匹配成对的括号
const parenthesisRegex = /(\([^)]*)\1\)/;
const text = '(text)(data)';
console.log(parenthesisRegex.test(text)); // true

// 3. 验证回文字符串
const palindromeRegex = /^(?<char>.).*(?=\k<char>$)/;
console.log(palindromeRegex.test('level')); // true
console.log(palindromeRegex.test('test')); // true

// 更精确的回文检测
function isPalindrome(str) {
  const reversed = str.split('').reverse().join('');
  return str === reversed;
}

console.log(isPalindrome('level')); // true
console.log(isPalindrome('hello')); // false
```

## 量词修饰

### 贪婪匹配

```javascript
// 默认情况下，量词是贪婪的（尽可能多地匹配）
const regex1 = /a.*b/;
const text1 = 'aab ab aab';

const result1 = regex1.exec(text1);
console.log(result1[0]); // 'aab ab aab' (尽可能多地匹配)

// 实际应用：提取引号内的内容（贪婪）
const quoteRegex1 = /"(.*)"/;
const quoteText1 = 'He said "hello world" and "goodbye"';

const match1 = quoteRegex1.exec(quoteText1);
console.log(match1[1]); // 'hello world" and "goodbye'

// HTML 标签匹配（贪婪）
const htmlRegex1 = /<div>.*<\/div>/;
const html1 = '<div>content1</div><div>content2</div>';

const match2 = htmlRegex1.exec(html1);
console.log(match2[0]); // '<div>content1</div><div>content2</div>'
```

### 非贪婪匹配

```javascript
// 使用 ? 使量词变为非贪婪（尽可能少地匹配）
const regex2 = /a.*?b/;
const text2 = 'aab ab aab';

const result2 = regex2.exec(text2);
console.log(result2[0]); // 'aab' (尽可能少地匹配)

// 实际应用：提取引号内的内容（非贪婪）
const quoteRegex2 = /"(.*?)"/;
const quoteText2 = 'He said "hello world" and "goodbye"';

const matches2 = quoteText2.matchAll(quoteRegex2);
console.log([...matches2].map(m => m[1])); // ['hello world', 'goodbye']

// HTML 标签匹配（非贪婪）
const htmlRegex2 = /<div>.*?<\/div>/g;
const html2 = '<div>content1</div><div>content2</div>';

const matches3 = html2.match(htmlRegex2);
console.log(matches3); // ['<div>content1</div>', '<div>content2</div>']

// URL 参数提取
const urlRegex = /\?(.*?)(?=#|$)/;
const url = 'https://example.com/path?name=value&age=25#section';

const params = urlRegex.exec(url);
console.log(params[1]); // 'name=value&age=25'
```

### 占有匹配（某些引擎支持）

```javascript
// 占有量词：*+, ++, ?+, {n,m}+ - 匹配但不回溯
const regex3 = /a++b/;
const text3 = 'aaab';

// 注意：JavaScript 对占有量词的支持有限

// 实际应用：避免回溯导致的性能问题
// 匹配 HTML 标签（更高效）
const efficientRegex = /<[^>]+>/g;
const html3 = '<div>content</div><span>text</span>';

const tags = html3.match(efficientRegex);
console.log(tags); // ['<div>', '</div>', '<span>', '</span>']
```

## 修饰符

### 全局匹配

```javascript
// g - 全局匹配
const regex1 = /hello/g;
const text1 = 'hello world hello';

const matches1 = text1.match(regex1);
console.log(matches1); // ['hello', 'hello']

// 不使用全局修饰符
const regex2 = /hello/;
const matches2 = text1.match(regex2);
console.log(matches2); // ['hello']

// 使用 exec 进行全局匹配
const regex3 = /hello/g;
const text3 = 'hello hello hello';

let result3;
while ((result3 = regex3.exec(text3)) !== null) {
  console.log(`找到匹配: ${result3[0]} 在位置 ${result3.index}`);
}

// 实际应用：统计单词出现次数
const wordRegex = /\b\w+\b/g;
const text4 = 'hello world hello test world';

const words = text4.match(wordRegex);
console.log(words.length); // 5

// 统计每个词的出现次数
const wordCount = {};
words.forEach(word => {
  wordCount[word] = (wordCount[word] || 0) + 1;
});

console.log(wordCount); // { hello: 2, world: 2, test: 1 }
```

### 忽略大小写

```javascript
// i - 忽略大小写
const regex4 = /hello/i;
const text4 = 'Hello HELLO hello';

const matches4 = text4.match(regex4);
console.log(matches4); // ['Hello']

// 全局 + 忽略大小写
const regex5 = /hello/gi;
const matches5 = text4.match(regex5);
console.log(matches5); // ['Hello', 'HELLO', 'hello']

// 实际应用：不区分大小写的匹配
const caseInsensitiveRegex = /javascript/gi;
const text5 = 'JavaScript is awesome. I love JavaScript.';

const matches6 = text5.match(caseInsensitiveRegex);
console.log(matches6); // ['JavaScript', 'JavaScript']

// 检查字符串中是否包含特定单词（不区分大小写）
function containsWord(text, word) {
  const regex = new RegExp(`\\b${word}\\b`, 'i');
  return regex.test(text);
}

console.log(containsWord('Hello World', 'hello')); // true
console.log(containsWord('Hello World', 'test')); // false
```

### 多行模式

```javascript
// m - 多行模式
const regex6 = /^hello/gm;
const text6 = `hello world
hello universe
hello there`;

const matches6 = text6.match(regex6);
console.log(matches6); // ['hello', 'hello', 'hello']

// 对比不使用多行模式
const regex7 = /^hello/g;
const matches7 = text6.match(regex7);
console.log(matches7); // ['hello'] (只匹配第一行的开头)

// 匹配每行的结尾
const regex8 = /world$/gm;
const text7 = `hello world
hello universe
test world`;

const matches8 = text7.match(regex8);
console.log(matches8); // ['world', 'world']

// 实际应用：解析 CSV 文件
const csv = `name,age,email
John,30,john@example.com
Jane,25,jane@example.com`;

const lines = csv.split('\n');
const headerRegex = /^([^,]+),([^,]+),([^,]+)$/;
const dataRegex = /^([^,]+),(\d+),([^@]+@[^.]+\.[a-z]+)$/;

lines.forEach((line, index) => {
  if (index === 0) {
    const header = line.match(headerRegex);
    console.log('表头:', header?.slice(1));
  } else {
    const data = line.match(dataRegex);
    console.log('数据:', data?.slice(1));
  }
});
```

### 点号匹配换行

```javascript
// s - dotAll 模式（ES2018），让 . 匹配包括换行符的所有字符
const regex9 = /hello.*world/s;

console.log(regex9.test(text8)); // true

const regex10 = /hello.*world/;

// 实际应用：匹配多行文本
const multiLineRegex = /<div>(.*?)<\/div>/s;
const html = `<div>line1
line2
line3</div>`;

const match = multiLineRegex.exec(html);
```

## 高级特性

### 环视断言组合

```javascript
// 组合使用各种断言
const complexRegex = /(?<=\{)\$\{(?!\s+)(?:(?!\$\{|\$\}).)*\}(?=\})/;

// 实际应用：匹配模板引擎的表达式
const template = 'Hello ${name}, your balance is ${balance} USD';
const expressions = template.matchAll(/\$\{[^}]+\}/g);

console.log([...expressions].map(m => m[0])); // ['${name}', '${balance}']

// 使用环视确保特定上下文
const contextRegex = /(?<=start:).*?(?=,|$)/;
const contextText = 'start:John,Doe,start:Jane,Smith';

const contexts = contextText.matchAll(contextRegex);
console.log([...contexts].map(m => m[0])); // ['John', 'Jane']
```

### 条件匹配

```javascript
// 某些正则引擎支持条件匹配 (?(id)yes|no)
// JavaScript 的支持有限，但可以通过替代方案实现

// 实际应用：根据是否存在匹配选择不同模式
function conditionalReplace(text) {
  // 如果文本包含 URL，使用 URL 模式，否则使用普通模式
  if (text.match(/https?:\/\//)) {
    return text.replace(/(https?:\/\/[^\s]+)/g, '<a href="$1">$1</a>');
  } else {
    return text;
  }
}

const text1 = 'Visit https://example.com';
const text2 = 'No URL here';

console.log(conditionalReplace(text1)); // 'Visit <a href="https://example.com">https://example.com</a>'
console.log(conditionalReplace(text2)); // 'No URL here'
```

### 原子组（某些引擎支持）

```javascript
// 原子组 (?>...) - 不可回溯的分组
// JavaScript 对原子组的支持有限，但可以通过其他方式实现类似效果

// 实际应用：防止过度回溯
const htmlRegex = /<div[^>]*>(?:[^<]*|<(?!\/div>))*<\/div>/;
const html = '<div>content</div><div>more content</div>';

const divs = html.match(htmlRegex);
console.log(divs); // ['<div>content</div>']
```

## 性能优化

### 避免灾难性回溯

```javascript
// 灾难性回溯示例
const badRegex = /^(a+)+$/;
const badText = 'a'.repeat(30) + '!';

console.time('bad');
const badResult = badRegex.test(badText);
console.timeEnd('bad'); // 可能很慢

// 优化的正则表达式
const goodRegex = /^a+$/;
console.time('good');
const goodResult = goodRegex.test(badText);
console.timeEnd('good'); // 很快

// 实际应用：避免嵌套量词
const optimizedRegex = /^a+$/;
```

### 使用字符类而非多个或

```javascript
// 低效
const inefficientRegex = /[a|b|c]/;

// 高效
const efficientRegex = /[abc]/;
```

### 预编译正则表达式

```javascript
// 不好的做法：每次都创建新的正则表达式
function test1(text) {
  return /hello/.test(text);
}

// 好的做法：预编译正则表达式
const HELLO_REGEX = /hello/;

function test2(text) {
  return HELLO_REGEX.test(text);
}

// 性能测试
const testText = 'hello world';

console.time('compiled');
for (let i = 0; i < 10000; i++) {
  test2(testText);
}
console.timeEnd('compiled');

console.time('uncompiled');
for (let i = 0; i < 10000; i++) {
  test1(testText);
}
console.timeEnd('uncompiled');
```

### 使用合适的方法

```javascript
// test() 方法只返回布尔值，性能最好
const regex = /hello/g;
const text = 'hello world hello';

if (regex.test(text)) {
  console.log('包含 hello');
}

// match() 返回匹配结果
const matches = text.match(regex);
console.log(matches);

// exec() 逐个匹配
let result;
while ((result = regex.exec(text)) !== null) {
  console.log(result[0], result.index);
}
```

### 5. 代码高亮

```javascript
// 简单的代码高亮
function highlightCode(code) {
  // 字符串
  code = code.replace(/(".*?"|'.*?'|`.*?`)/g, '<span class="string">$1</span>');
  
  // 关键字
  const keywords = ['function', 'const', 'let', 'var', 'return', 'if', 'else', 'for', 'while'];
  const keywordRegex = new RegExp(`\\b(${keywords.join('|')})\\b`, 'g');
  code = code.replace(keywordRegex, '<span class="keyword">$1</span>');
  
  // 数字
  code = code.replace(/\b(\d+)\b/g, '<span class="number">$1</span>');
  
  // 注释
  code = code.replace(/(\/\/.*$)/gm, '<span class="comment">$1</span>');
  
  return code;
}

const code = `function test() {
  const x = 10;
  // This is a comment
  return x + 5;
}`;

console.log(highlightCode(code));
```

## 注意事项

1. **复杂度控制**：避免过于复杂的正则表达式，必要时拆分为多个步骤。

2. **回溯问题**：注意嵌套量词可能导致灾难性回溯，影响性能。

3. **浏览器兼容性**：某些高级特性在不同浏览器中的支持程度不同。

4. **可读性**：复杂的正则表达式应该添加注释，或拆分为多个部分。

5. **边界情况**：测试各种边界情况，包括空字符串、特殊字符等。

6. **转义字符**：注意特殊字符的转义，特别是在动态构建正则表达式时。

7. **性能优先**：在性能敏感的场景中，优先选择更简单、更高效的正则表达式。

8. **错误处理**：正确处理正则表达式匹配失败的情况。

## 最佳实践

1. **命名捕获组**：使用命名捕获组提高代码可读性。

2. **非贪婪匹配**：默认使用非贪婪量词，避免过度匹配。

3. **预编译正则**：将正则表达式预编译为变量，提高性能。

4. **分解复杂表达式**：将复杂的正则表达式分解为多个简单的部分。

5. **添加注释**：为复杂的正则表达式添加清晰的注释。

6. **编写测试**：为正则表达式编写全面的测试用例。

7. **使用原始字符串**：使用字符串模板或转义避免双重转义问题。

8. **定期优化**：定期审查和优化正则表达式，确保性能和准确性。

## 总结

正则表达式进阶技巧是提升字符串处理能力的关键。通过掌握捕获组、零宽断言、反向引用、量词修饰等高级特性，我们能够处理更复杂的文本操作。

- **充分测试**：编写全面的测试用例验证正确性
